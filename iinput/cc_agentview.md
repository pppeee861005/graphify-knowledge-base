這份來源介紹了 **Claude Code** 最新的 **Agent View** 功能，這是一個能顯著提升開發效率的**後端任務管理界面**。透過此功能，使用者可以同時執行**多個獨立的 AI 會話**，且無需手動切換終端機視窗，即可在背景處理多項專案任務。系統利用**顏色與圖示標記**清楚顯示各個任務的運行狀態，例如綠色代表完成、黃色則表示需要用戶輸入。使用者能透過 **Peak 功能**快速預覽任務進度，或是利用 **Attach 模式**深入查看完整的對話細節。這項更新解決了傳統操作中頻繁切換界面的困擾，讓開發者能以**並行處理**的方式大幅節省每日的工作時間。

使用 **Agent View** 管理多個背景任務，可以讓你同時處理多個 AI 對話而無需手動切換視窗。以下是根據來源整理的實戰操作指南：

### 1. 進入與啟動任務
*   **版本檢查**：確保你的版本在 **2.1.139** 以上。
*   **進入指令**：在 Terminal 輸入 `claude agents` 即可進入主介面，這是一個專門管理所有背景對話（session）的視窗。
*   **啟動背景任務**：直接在底部的輸入框輸入指令（例如：要求整理文件摘要），系統會立刻產生一個新的任務列。
*   **從外部啟動**：如果你正在執行其他指令，也可以使用 `-b` 參數（例如 `claude -b "指令"`）直接啟動背景任務，系統會回傳一個短 ID，該任務會同步出現在 Agent View 中。

### 2. 監控任務狀態
Agent View 透過圖標顏色與形狀來顯示任務進度：
*   **旋轉圖示**：代表任務**正在執行中**。
*   **黃色**：代表任務**需要你的輸入**（Needs Input），例如 AI 有問題想確認。
*   **綠色**：代表任務已**順利完成**。
*   **紅色**：代表任務**發生錯誤**。
*   **灰色**：代表任務由**使用者自行停止**。
*   **暗色/閒置**：代表目前任務處於**閒置狀態**。

此外，圖標的形狀也有意義：**實心**表示後台進程仍活躍，可直接回覆；**小原點**表示程序已結束但可重新接續；**空心**則代表正在兩次疊代之間的休息時間。

### 3. 管理與切換任務
*   **快速預覽 (Peek)**：選中任務後按 **空白鍵 (Space)**，右側會跳出 **Peek Panel**。你可以直接看到該任務最近的進度、是否需要回覆或有無開 PR，無需切換進去。
*   **完整進入 (Attach)**：選中任務後按 **Enter**，可以完整切換進該對話進行細節操作。
*   **退出對話**：在 Attach 模式下，按 **Ctrl + C** 即可退回到 Agent View 介面，而該對話仍會在背景繼續運行或保持狀態。
*   **並行處理**：你可以同時啟動多個任務（如同時處理三個不同的程式碼修改），各任務會實時刷新狀態，互不干擾。

透過這種方式，即使你退出了 Claude 或重開 Terminal，背景任務依然會持續運行，直到完成或出錯為止。

根據來源資料，在 Agent View 中是支援**多個任務同時運行**的。

以下是關於同時運行任務數量的具體資訊：

*   **多任務並行**：來源明確提到背景 Session 支持多個任務同時進行，且每個任務完成時都會實時刷新列表。
*   **實測範例**：在影片演示中，作者示範了**同時安排 3 個新的 Session** 進行處理，並且這三個任務都能在背景同時執行，無需使用者介入。
*   **效率參考**：雖然來源中沒有明確提到硬體的上限數值，但影片標題指出透過這種方式可以「**同時跑 10 個 AI**」，從而達到大幅節省時間的效果。

當多個任務同時運行時，Agent View 會為每個任務顯示獨立的一行（Row），你可以透過最左側的圖標狀態（如旋轉圖示代表正在執行）來監控所有任務的即時進度。

根據來源資料，Agent View **可以監控背景任務是否開啟了 PR（Pull Request）**，但具體是否「自動提交」取決於你下達給 AI 的指令以及任務的執行流程。

以下是關於背景任務與 PR 管理的詳細說明：

*   **查看 PR 狀態**：在 Agent View 介面中，你不需要進入（Attach）特定的對話，只需選中該任務並**按下「空白鍵」（Space）**開啟 **Peek Panel**。這個預覽面板會直接顯示該 Session 最近的進度，包含是否需要你回覆，以及**「有沒有開 PR」**。
*   **任務執行與回報**：背景任務（Background Session）會依照你的指令自動運行。如果你的指令要求 AI 完成程式碼修改並提交 PR，AI 會在後台嘗試完成這些步驟。當任務完成後，狀態列會顯示為 **Completed（綠色）**，你可以在 Peek Panel 或進入對話中確認 PR 的詳細資訊。
*   **多工管理優勢**：由於 Agent View 支持**多個任務同時運行**，你可以同時啟動多個涉及程式碼修改的 Session，並透過 Peek Panel 快速掃視哪些任務已經成功建立了 PR，而無需手動切換多個 Terminal 視窗。

總結來說，Agent View 提供了一個集中的介面讓你**快速確認 PR 的開啟狀況**，讓你能更有效率地管理多個 AI 代理人的工作產出。
