$ graphify query '黑箱 三層次 工具論'
✓ 預期輸出：相關文章、概念節點、共現頻率圖

Task 1.2：試驗 graphify query（第二個查詢）

目標：查看「主權 AI 人類介入」的關聯網絡
執行時間：5-10 分鐘
在哪執行：Claude Code Terminal
$ graphify query '主權 AI 人類介入'

在哪執行：Claude Code Editor

✓ 輸出：一個有 5-7 個核心章節的故事框架

目標：用你的筆，基於 Hermes 準備好的素材，寫出 3000-4000 字正文
執行方式：Claude Code Editor 實時編輯，反覆迭代
✓ DK_S01E02_黑箱的三個層次_draft.md（完成版）
Day 2 架構

並行階段（Task 2.0-2.5）： Hermes 同時跑 6 個任務，你去睡覺或做別的。
等待階段（Task 2.6）： Task 2.6 被 Task 2.1-2.5 gate 了，自動等待。
人類親自階段（Task 2.7）： 當 Hermes 準備好所有素材，你上場寫正文。

關鍵點：如果你在寫到一半想改大綱（比如「第二層加一個小節」），直接在 Claude Code 裡改，不用回 Kanban 重來。

Day 3

Agent View 驗收

讀稿、調整措辭、最後的知識圖譜更新

Task 3.1：打開草稿、讀一遍

執行環境：Claude Code Editor
目標：打開 DK_S01E02_黑箱的三個層次_draft.md，用 Agent View 讀全文
執行時間：15-20 分鐘
檢查項目：
  • 敘事連貫性（是否延續 E01 的風格）
  • 論證完整性（三層黑箱是否足夠有說服力）
  • 有無重複或冗餘段落
✓ 讀稿完畢，列出修改清單

Task 3.2：編輯與微調

執行環境：Claude Code Editor 的 inline editing
目標：
  • 修改句式（更流暢的措辭）
  • 補充例子（如果三層黑箱的某層舉例不夠具體）
  • 刪除冗餘（如發現重複論點）
執行時間：20-30 分鐘
✓ 草稿 → 精製版

Task 3.3：graphify update（最終更新）

執行環境：Claude Code Terminal
目標：新文章 E02 寫完了，把它加入圖譜
執行時間：2-5 分鐘
$ graphify update .
✓ 圖譜已更新，E02 現在被索引，下次 query 會找到它的關聯

Task 3.4：存檔確認

檔案位置：系列文章/暗知識系列/DK_S01E02_黑箱的三個層次_draft.md
檢查項：
  • 字數是否在 3000-4000 字
  • 繁體中文編碼是否正確
  • markdown 格式無誤
✓ E02 完成，可交付
Day 3 小結與知識飛輪

你的角色：編輯器。從作者變成編輯，讀、改、定版。
Kanban 的角色：退場。Day 3 不再需要後臺任務，只有你的實時決策。
知識飛輪啟動：

  • S1E01 寫完 → graphify update
  • S1E02 寫完 → graphify update
  • 下次寫 S1E03 時，graphify query 會返回 E01 + E02 的關聯
  • 敘事連貫性自動提升 ✨
