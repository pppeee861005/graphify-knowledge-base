Task 1.1：第一個 graphify 查詢

目標：探索「黑箱 三層次 工具論」的關聯網絡

1

打開 Claude Code terminal

在你的機器上打開 Claude Code，準備執行指令

2

進入暗知識系列目錄

先切到正確的目錄

cd ~/系列文章/暗知識系列
💡 如果路徑不同，改成你實際存放暗知識文章的位置
3

執行第一個 graphify 查詢

查詢「黑箱、三層次、工具論」這三個核心概念的關聯

graphify query '黑箱 三層次 工具論'
✓ 預期輸出：
• 相關文件清單
• 概念節點及其出現頻率
• 概念之間的語義距離
• 建議的論述脈絡
4

檢查輸出

截圖或複製結果，記下哪些文章被提及

✓ Task 1.1 完成！你現在知道了「黑箱 三層次 工具論」在整個系列中的關聯

Task 1.2：第二個 graphify 查詢

目標：探索「主權 AI 人類介入」的關聯網絡

1

執行第二個 graphify 查詢

查詢另一個核心角度：主權與人類決策權

graphify query '主權 AI 人類介入'
✓ 預期輸出：
• 涉及「主權」的文章
• 涉及「人類介入」的邏輯線
• 跨文章的倫理論述框架
• 與第一個查詢的交叉點
2

對比兩個查詢結果

在 Claude Code 的 editor 裡開一個新文件，記下兩個查詢的共同點和差異

# 在 editor 裡手寫一份比較： 查詢 1：黑箱 三層次 工具論 - 重點文檔：[列出] - 核心視角：[機制層面] 查詢 2：主權 AI 人類介入 - 重點文檔：[列出] - 核心視角：[倫理 & 權利層面] 交叉點： - [共同出現的概念] - [可能的敘事連接]
✓ Task 1.2 完成！你現在有了兩個切面的全景圖

Task 1.3：手寫初版大綱

目標：基於兩個查詢，構思 E02 的故事框架

1

在 Claude Code editor 裡新建大綱文件

創建 DK_S01E02_初版大綱.md

# 暗知識系列 S1E02：黑箱的三個層次 — 初版大綱 ## 開場（500 字） [延續 E01 的敘事風格，說明為什麼要深入「黑箱」] ## 第一層：技術黑箱（700 字） [工具、算法、代碼層面的不透明性] ## 第二層：認知黑箱（700 字） [人類理解、心智模型、感知偏差] ## 第三層：主權黑箱（700 字） [誰有決策權、AI 與人類的權力邊界] ## 結語（300 字） [三層黑箱如何匯聚成「新的人類困境」] ## 註： - 字數目標：3000-4000 字 - 風格：延續 E01 的「引入-深化-反思」模式 - 素材來源：待 Day 2 補充
2

細化大綱的每個章節

在編輯器裡逐一展開，寫出每個章節的 3-5 個核心論點

# 第一層：技術黑箱 — 細化 ## 1.1 什麼是技術黑箱？ - 定義：我們使用的工具（AI、算法、應用）的內部機制不可見 - 例子：[你決定在這裡加什麼例子？] ## 1.2 技術黑箱的三個表現 - a) 代碼黑箱 - b) 模型黑箱 - c) 組織黑箱 ## 1.3 我們面臨什麼問題？ - [從日常體驗出發] ## 1.4 小結 - [承上啟下]
3

保存初版大綱

存成 Markdown 檔案，為 Day 2 準備
