# Kanban 流程總覽

## 任務依賴流程

```
T1 (Researcher) ──┐
                  ├──→ T3 (Analyst) ──→ T4 (Writer)
T2 (Researcher) ──┘
```

**特點：** T1 + T2 並行執行，T3 等待兩者完成，T4 最後上場撰寫

## Kanban 運轉機制

| 機制 | 說明 |
|------|------|
| **並行處理** | T1 + T2 同時由兩個 Researcher 執行 |
| **依賴門控 1** | T3 自動等待 T1 ✓ + T2 ✓，完成後自動轉 ready 狀態 |
| **依賴門控 2** | T4 自動等待 T3 ✓，完成後自動轉 ready 狀態 |
| **Gateway 調度** | 每 60 秒自動掃描，將 ready task 分配給對應 assignee |
| **工作區隔離** | 每個 task 有獨立的 scratch 目錄，避免干擾 |
| **審計追蹤** | SQLite 永久記錄 event log + comment thread（可回溯） |

## 時間統計

| 階段 | 耗時 | 說明 |
|------|------|------|
| **T1 + T2（並行）** | 1 分鐘 | 兩個任務同時進行 |
| **T3（合成分析）** | 7 分鐘 | 整合前兩個任務的產出 |
| **T4（寫作）** | 4 分鐘 | 根據 T3 指引撰寫全文 |
| **總耗時** | **12 分鐘** | 從開始到草稿完成 |

**關鍵優勢：** 通過並行 + 自動化門控，將串行時間壓縮到最短

## 核心亮點

✨ **並行優化**：T1 + T2 同時執行，而不是串行
✨ **智能分工**：Researcher 收集素材 → Analyst 規劃結構 → Writer 執行撰寫
✨ **自動化流程**：依賴關係自動管理，無需人工干預
✨ **可審計性**：所有操作步驟完整記錄，可回溯驗證
