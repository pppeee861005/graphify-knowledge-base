🎯 我推薦方式 1（API Key）用於 Vultr
最直接、最快、最適合 headless VPS 環境。以下詳細步驟都是針對 API Key 方式。

詳細設定步驟（API Key 方式）

第一部分：本地機器上的準備

1

登入 Anthropic Console

生成 API Key

訪問 console.anthropic.com

https://console.anthropic.com
需要 Anthropic Console 帳戶。如果只有 Claude Pro，需要升級到 Console（按使用量付費）
2

建立 API Key

左邊選單 → API Keys → Create Key

複製產生的 Key，格式應該是：sk-ant-api03-...

✓ 保存這個 Key 到安全的地方（密碼管理器或加密檔案）

第二部分：Vultr VPS 上的設定

3

SSH 進入 Vultr VPS

用 root 或你的 SSH 密鑰

ssh root@your-vultr-ip
💡 將 your-vultr-ip 替換為你的實際 IP
4

更新系統

確保所有軟體包是最新的

apt update && apt upgrade -y
5

安裝 Node.js（如果用 npm 方式）

原生安裝器不需要，但建議裝以防萬一

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt-get install -y nodejs
✓ 驗證：node --version
6

安裝 Claude Code

原生安裝器方式

curl -o- https://api.claude.sh/install.sh | bash
💡 安裝器會把 claude 放在 ~/.local/bin/claude
✓ 驗證：claude --version
7

設定 API Key 環境變數

最簡單和推薦的方式

臨時設定（此 session 有效）：

export ANTHROPIC_API_KEY="sk-ant-api03-your-actual-key-here"

永久設定（建議，加到 ~/.bashrc）：

echo 'export ANTHROPIC_API_KEY="sk-ant-api03-your-actual-key-here"' >> ~/.bashrc source ~/.bashrc
⚠️ 重要：用你從 Console 生成的實際 API Key，不要用示例文字
8

驗證認證

運行診斷工具

claude doctor
✓ 你應該看到：
Authentication: ✓ Complete
Node.js version: ✓ v22.x.x
System: ✓ Linux Ubuntu 22.04
9

測試 Claude Code

運行一個簡單命令確認工作

cd ~ claude -p "What's the current date and time?"
✓ 如果看到 Claude 的回覆，認證成功！

安全加固清單（必做）

⚠️ VPS 會在上線後數分鐘內遭到掃描
如果沒做安全加固，風險非常高。以下步驟 5 分鐘完成，極大降低被攻擊的機率。
1

建立非 root 使用者

不要用 root 執行 Claude Code

adduser claude usermod -aG sudo claude
2

設定 SSH 密鑰認證

禁用密碼認證

mkdir -p /home/claude/.ssh # 從你的本地機器複製公鑰： ssh-copy-id claude@your-vultr-ip
💡 在你的本地機器上運行，假設你有 SSH 密鑰對
# 禁用密碼認證 sudo sed -i 's/PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config sudo systemctl restart sshd
3

啟用防火牆（UFW）

允許 SSH，其他全部阻擋

sudo ufw default deny incoming sudo ufw default allow outgoing sudo ufw allow OpenSSH sudo ufw enable
4

安裝 Fail2Ban（自動擋暴力攻擊）

自動封禁多次失敗登入的 IP

sudo apt install fail2ban -y sudo systemctl enable fail2ban sudo systemctl start fail2ban
5

API Key 安全儲存

不要在代碼或 .bash_history 中洩露

# 建立安全的配置檔案 sudo mkdir -p /etc/claude-code sudo tee /etc/claude-code/.env > /dev/null << 'EOF' ANTHROPIC_API_KEY="sk-ant-api03-your-key" EOF sudo chmod 600 /etc/claude-code/.env sudo chown claude:claude /etc/claude-code/.env
# 在 ~/.bashrc 中引用（不要直接寫 key） source /etc/claude-code/.env
6

使用 tmux 保持會話（可選但推薦）

SSH 斷線後 Claude Code 繼續運行

sudo apt install tmux -y # 啟動新的 tmux session tmux new -s claude-session # 在 tmux 內執行 Claude Code cd ~/my-project claude # 分離（Ctrl+B，然後 D） # 重新連接： tmux attach -t claude-session

安全驗證檢查清單

  • ☐ SSH 密鑰認證已設定，密碼認證已禁用
  • ☐ UFW 防火牆已啟用，僅允許 OpenSSH
  • ☐ Fail2Ban 已安裝並執行
  • ☐ API Key 儲存在 /etc/claude-code/.env（600 權限）
  • ☐ 非 root 使用者已建立，具有 sudo 權限
  • ☐ 運行 claude doctor 確認一切正常
