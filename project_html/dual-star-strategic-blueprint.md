## 第一層：真實的系統層次

```
┌─────────────────────────────────────────────────────────────┐
│                      用戶引力漏斗                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 內容發現      2. 體驗互動      3. 訂閱決策   4. 變現  │
│   (被動看)         (主動玩)         (邮件)       (付費)   │
│                                                             │
│   Web互動頁面  ←→  Substack系統   ←→  產品售賣           │
│   (無轉化)        (有轉化)         (有營收)      │
```

### Substack 的真實角色：三層

| 層級 | 功能 | 執行 | 價值 |
|-----|------|------|-----|
| **L3: 變現層** | 訂閱費 + 贊助 + 廣告 | Substack 官方 | 💰 直接營收 |
| **L2: 數據層** | CRM + 郵件分組 + 行為追蹤 | Substack API | 📊 決策情報 |
| **L1: 發布層** | 文章編輯 + 版本控制 + 檔案託管 | Substack 編輯器 | 📝 內容載體 |

## 核心洞察

**為什麼這個模式會工作？**

1. **Substack = CRM 基礎**
   - 你的資產不是 Web 的訪客（隨時會流失）
   - 你的資產是訂閱者郵件列表（可反復觸達、可變現）

2. **Web = 引力引擎**
   - Substack 的內容排版已經是「舊時代審美」
   - 互動網頁是「新時代內容體驗」
   - 用 Web 吸引用戶，用 Substack 留住用戶

3. **變現 = 多元收入**
   - 不依賴單一平台 (Substack 分成太低)
   - Substack MRR 是基礎，聯動產品是增長
   - 目標：Substack 做「客戶管理」，Gumroad 做「商品管理」

4. **品牌積累 = 超級個體**
   - 每個訂閱者 = 你的個人「直播間」
   - 每個診斷用戶 = 你的潛在客戶
   - 社群 = 你的「員工」（他們幫你傳播）
