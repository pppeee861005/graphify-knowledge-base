## 概念層

### 第 1 篇｜蜂群 Agent 是什麼？運算資源的新範式

**發布時機：** Week 1｜定錨篇，建立讀者認知框架
**核心命題：** Agent 從「智慧寵物」進化為「可拋棄的函數呼叫」

**大綱：**
1. 開篇衝擊：Netflix 如何用蜂群 Agent 把基礎設施問題從小時縮到分鐘
2. 定義：什麼是蜂群 Agent？與傳統 Agent 的本質區別
3. 六大核心特性
   - 按需湧現與銷毀——毫秒級啟動，執行完即毀滅
   - 故障容錯——單蜂故障，毫秒級生成新蜂替代
   - 異構 LLM 調度——每隻單蜂按需加載適合的模型
   - 無狀態單蜂——純計算函數，無記憶無持久性
   - Session 統一調度——狀態、記憶、工具由持久層統管
   - 可適應性——隨新 LLM 迭代自動進化（第 2 篇展開）
4. 核心比喻：Lambda 函數 × 人類智慧 = 蜂群
5. 收尾問題：如果 Agent 不需要意識，人類的價值在哪裡？

### 第 2 篇｜可適應性：蜂群為什麼永遠不會被淘汰

**發布時機：** Week 2｜深化篇，引導讀者重新評估「什麼是長期資產」
**核心命題：** 軟體是「凍結的決策」；蜂群是「流動的智慧」

**大綱：**
1. 開篇反問：你的系統能在 Anthropic 發布新模型後，5 分鐘內自動升級嗎？
2. 傳統軟體的困境：決策凍結在發布那天
   - 代碼邏輯固化，能力上限確定
   - 更新 = 重新測試 = 風險 = 成本
3. 蜂群的進化機制
   - 邏輯層（人類決策）與執行層（LLM）完全分離
   - 新模型發布 → 只更新執行層 → 零停機、零風險
4. 六個時間尺度的進化
   - 小時級：新模型版本自動可用
   - 日級：Lead Agent 收集軌跡，自動優化分工
   - 週級：新能力試驗與灰度切換
   - 月級：架構滾動遷移
   - 季度級：新模態 Agent 無縫添加
   - 年級：AGI 到來，蜂群直接集成
5. 對比案例：Zapier（規則固化）vs 蜂群版本自動化
6. 核心命題：設計是永恆資產，代碼會老化

## 架構層

### 第 3 篇｜解耦哲學：Anthropic 如何設計蜂群的靈魂

**發布時機：** Week 3｜技術篇，給工程師讀者的架構深潛
**核心命題：** 為「尚未被想到的程式」而設計

**大綱：**
1. 開篇：作業系統如何解決「為未來程式設計」的問題
2. Anthropic 的三層分離架構
   - 大腦（Brain）：只負責推理與決策
   - 框架（Harness）：呼叫模型、路由工具、異常處理
   - 執行（Session）：持久層，事件日誌，統一調度
3. Harness 的假設會過時，Session 的接口永存
   - 案例：Claude Sonnet 4.5 的「上下文焦慮」在 Opus 4.5 消失
   - 同一套 Harness，無需改動，自動享受模型升級
4. Session 的核心設計：getEvents() 接口
   - 事件日誌 vs 傳統記憶體
   - 任何新的蜂都可以讀取過去所有狀態
   - 完全並行化成為可能
5. 類比：read() 指令從磁碟包到 SSD 都不需要改
6. Netflix 案例：多 Agent 編排如何解決基礎設施問題
7. 對開發者的啟示：你現在寫的 Agent 邏輯，10 年後還能用嗎？

### 第 4 篇｜蜂群設計模式：四種架構讓你用對場景

**發布時機：** Week 4｜實戰篇，附架構圖與成本計算
**核心命題：** 選錯模式，蜂群的效益減半

**大綱：**
1. 開篇：為什麼「蜂群設計」比「模型選擇」更重要
2. 模式一：扇出-扇入（Fan-out / Fan-in）
   - 定義：Lead Agent 分解 → N 個並行蜂 → 結果彙總
   - 適用場景：批量資料處理、內容批量生成、並行資料收集
   - 成本計算：N 倍並行，時間縮 N 倍，成本持平
   - 實例：100 張平面圖批量轉 DXF（聯盟實驗）
3. 模式二：流水線（Pipeline）
   - 定義：A → B → C，後級依賴前級輸出
   - 適用場景：多步驟工作流，有依賴的處理鏈
   - 變體：流水線 + 扇出組合
   - 實例：寫作 → 編輯 → 審查 → 發布
4. 模式三：樹形決策（Tree Search）
   - 定義：Lead Agent 生成假設 → 並行驗證 → 遞歸深化
   - 適用場景：複雜決策、創意探索、問題診斷
   - 實例：投資決策的多角度評估
5. 模式四：動態生成（Dynamic Generation）
   - 定義：Lead Agent 按任務複雜度彈性決定蜂群規模
   - 適用場景：任務複雜度不均勻的批量處理
   - 關鍵技術：複雜度評分 → 動態分配資源
6. 選擇指南：四個問題決定用哪個模式
7. 成本對比表：四種模式在不同場景下的效益

### 第 5 篇｜異構 LLM 調度：讓蜂群用最少的錢做最好的事

**發布時機：** Week 5｜效率篇，企業最在意的成本問題
**核心命題：** 選錯模型，成本貴 5–10 倍

**大綱：**
1. 開篇衝擊：同樣的任務，用 Opus 比用 Haiku 貴 50 倍
2. 三層 LLM 的分工邏輯
   - Haiku：簡單任務（翻譯、格式化、分類）
   - Sonnet：中等任務（邏輯推理、內容生成）
   - Opus：複雜任務（多步推理、戰略決策）
3. Lead Agent 如何動態判斷任務複雜度
   - 複雜度評分系統設計
   - 邊界案例：模糊任務如何處理
4. 自適應調度的實現
   - 靜態配置 vs 動態配置
   - 灰度試驗：部分蜂群使用新配置
5. 成本計算實例
   - 100 個任務，全用 Opus vs 異構調度
   - 預期成本下降 50–70%
6. 聯盟實驗數據：三個蜂群實驗的成本對比（cad-architect / 文案評估 / 銷售分析）
7. 對企業的建議：如何設計你的第一套 LLM 調度規則

## 預測層

### 第 6 篇｜Dreaming：蜂群如何在睡眠中自我進化

**發布時機：** Week 6｜哲學篇，蜂群的「潛意識」
**核心命題：** 蜂群學習的單位不是個體，而是集體

**大綱：**
1. 開篇類比：人類睡眠如何鞏固記憶、提取模式
2. Anthropic Dreaming 的機制
   - 排程流程：審視過去會話 → 提取模式 → 更新記憶
   - 識別：反覆錯誤、聚合的工作流、跨團隊偏好
   - 控制權：自動更新 vs 人工審核
3. 集體學習 vs 個體學習
   - 傳統 Agent：只能從自己的對話中學習
   - Dreaming 蜂群：從所有蜂的執行軌跡中學習
   - 湧現效應：100 個蜂的集體經驗 > 1 個有記憶的 Agent
4. 哲學維度
   - 無意識（個體蜂）+ 集體記憶（Dreaming）= 蜂群意識？
   - 這是 AGI 的前兆，還是只是更聰明的快取？
5. 聯盟實驗：Dreaming 對蜂群輸出品質的改善幅度
6. 對開發者的啟示：如何設計「值得被 Dreaming 學習」的執行軌跡
7. 收尾：當蜂群在夜間學習，明天的你將面對一個更強的協作夥伴

### 第 7 篇｜SaaS 死亡名單：蜂群將在 12 個月內摧毀哪些產業

**發布時機：** Week 7｜預測篇，病毒傳播潛力最高
**核心命題：** 蜂群的三重碾壓：速度、成本、可適應性

**大綱：**
1. 開篇框架：SaaS 死亡的三個判斷標準
   - 商業模式建立在人頭 / 工時 / 座位計費
   - 核心功能是資訊搬運或流程重複
   - 切換成本被蜂群拉到零
2. 高危名單逐一分析
   - Zapier / Make（自動化工作流）：規則引擎 vs 動態決策
   - Copy.ai / Jasper（文案生成）：單一視角 vs 多角度蜂群
   - Salesforce Einstein（CRM 分析）：靜態模型 vs 實時蜂群
   - Zendesk / Intercom（客服）：排隊等待 vs 蜂群瞬間應答
   - Workday / BambooHR（人力資源）：表單流程 vs 動態評估蜂群
3. 死亡時間表：12–36 個月的衰退曲線
4. 誰會活下來？
   - 基礎設施層（Anthropic、Stripe、Databricks）
   - 無法標準化的服務（法律、心理、高端設計）
   - 轉型成功的垂直 Agent 平台
5. 新贏家的樣貌：不賣「軟體」，賣「AI 員工」
6. 對企業的行動建議：現在該做什麼準備

### 第 8 篇｜新人類聯盟如何駕馭蜂群：Homo Coalitio 的戰略藍圖

**發布時機：** Week 8｜品牌收尾篇，聯盟宣言
**核心命題：** 人類 + 蜂群的共生體，才是蜂群時代的真正贏家

**大綱：**
1. 開篇回顧：8 篇系列建立了什麼認知框架？
2. 聯盟的三重定位
   - 知識供應商：解讀 Anthropic 生態演變，產出戰略洞察
   - 生態觀察者：監測 Managed Agents / Dreaming / 多 Agent 編排
   - 蜂群架構師：設計可複製的蜂群模式，積累不老化的資產
3. 四層戰略的實踐
   - 監測層：每週信號追蹤，搶先解讀生態變化
   - 實驗層：三個蜂群實驗，軌跡數據積累
   - 知識層：設計模式庫，Cookbook 發布
   - 預測層：SaaS 死亡預測，話語權建立
4. 給讀者的行動建議：如何在蜂群時代找到自己的位置
   - L1（執行者）：學習使用蜂群工具
   - L2（設計者）：掌握蜂群設計模式
   - L3（架構師）：成為垂直領域的蜂群設計專家
   - L4（聯盟者）：人 + 蜂群的 Homo Coalitio
5. 哲學收尾：三個永恆問題在蜂群時代的新答案
   - 生命是否可化約為演算法？
   - 意識 vs 智能，孰輕孰重？
   - 無意識演算法比人類更懂人類時，社會如何轉變？
6. 結語：蜂群不是終點，而是 Homo Coalitio 的起點
