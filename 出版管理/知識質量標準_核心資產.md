## 📋 每篇文章的質量檢查清單

發布前，每篇文章必須達成以下標準：

### **A. 內容質量（必須項）**

#### **1. 系統性框架（60分）**
```
✅ 該篇是否有完整的知識框架？
   └─ 而非單點論述或碎片觀點

✅ 框架是否「新」？
   └─ 不是在複述既有常識

✅ 框架的層次清晰嗎？
   └─ 讀者能跟著邏輯走
   └─ 而非迷失在細節裡

實例（達標）：
├─ 「三樣失竊物」（注意力、創造力、自我效能感）✅
├─ 「引力轉移理論」（消費 vs 創造的物理學）✅
└─ 「Agent架構的五大設計原則」（待驗證）

實例（不達標）：
├─ 「AI很厲害」（無框架）❌
├─ 「我今天讀了..感想是...」（碎片）❌
└─ 「Software 3.0 改變世界」（無深度論證）❌
```

#### **2. 獨特視角（20分）**
```
✅ 這個觀點在市場上有多少人說過？
   ├─ 0人說過 = 100% 原創 ⭐⭐⭐⭐⭐
   ├─ 1-2人說過 = 80% 原創 ⭐⭐⭐⭐
   ├─ 3-5人說過 = 50% 原創 ⭐⭐⭐
   └─ 10人以上說過 = 停止寫這個話題

✅ 即使主題不新，「角度」是否新？
   └─ 比如「AI創作」已有人談
   └─ 但「蜂群Agent的分佈式創意」是新角度 ✅

實例（高獨特性）：
├─ 「蜂群Agent vs 單一Agent的組織論」✅
├─ 「Software 3.0 的散戶基礎設施」✅
└─ 「從毒癮視角看創造力的喚醒」✅

實例（低獨特性）：
├─ 「AI改變了世界」❌
├─ 「短視頻很成癮」❌
└─ 「需要提升自律」❌
```

#### **3. 實質洞察 + 案例支撐（20分）**
```
✅ 是否有「讀者沒想到」的洞察？
   ├─ 而非他們已經知道的常識
   └─ 讀完後應該有「啟蒙」感

✅ 是否有具體案例或數據佐證？
   ├─ 抽象理論需配合具體例
   ├─ 比如「神經科學的耐受性提升」+「TikTok的7秒鉤子」
   └─ 不是空洞的哲學碎碎念

✅ 案例是否足夠有說服力？
   └─ 是讀者能理解和認同的現象

實例（高洞察）：
├─ 「演算法不是技術，是武器」（新視角 + 心理學案例）✅
├─ 「心流 vs 多巴胺速食」（神經科學 + 行為比較）✅
└─ 「宜家效應證明自己做的東西更珍貴」（實驗驗證）✅
```

### **B. 內容量（必須項）**

```
字數範圍：800-2000 字
├─ 最少 800字（太短無法深展框架）
├─ 最多 2000字（超過讀者注意力跨度）
└─ 最佳 1200-1500字（深度 + 可讀性平衡）

段落結構：3-5 個核心段落
├─ 開篇：設置問題或情景（50-100字）
├─ 主體：3-4 個核心觀點（每個 200-400字）
└─ 結尾：洞察總結 + 後續預告（100-150字）
```

### **C. 邏輯清晰度（必須項）**

```
✅ 邏輯是否環環相扣？
   ├─ A 導向 B，B 導向 C，而非跳躍式
   └─ 讀者能跟著思路走

✅ 句子是否簡潔有力？
   ├─ 平均句長 15-20 字（中文）
   └─ 避免「堆砌修飾詞」的冗長句

✅ 過渡是否自然？
   └─ 段落間有邏輯銜接，而非硬轉
```

### **D. 後續預告（加分項）**

```
✅ 是否在結尾預告「下一篇」？
   └─ 「引導訂閱」的關鍵

✅ 預告是否誘人？
   ├─ 而非「敷衍式預告」
   ├─ 比如「第三篇會講..」❌
   └─ 而是「為什麼創造比消費更上癮？」✅
```

## 🚫 質量不合格的跡象

如果文章出現以下特徵，**必須推遲發布**：

| 紅旗 | 嚴重程度 | 補救方案 |
|------|---------|---------|
| 沒有清晰的框架，只有感想 | 🔴 嚴重 | 返工重構，至少推遲 1 週 |
| 觀點在網上已被廣泛討論 | 🔴 嚴重 | 找新角度或放棄此話題 |
| 案例不足，全是理論 | 🟡 中度 | 補充 2-3 個具體案例 |
| 邏輯跳躍，讀者容易迷失 | 🟡 中度 | 重寫段落過渡，加入橋接句 |
| 字數不足（<700字） | 🟡 中度 | 補充內容至 800+ |
| 標題平淡，無吸引力 | 🟢 輕度 | 優化標題（加入對比/新概念） |
| 有小錯誤（OCR、標點） | 🟢 輕度 | 最後複審修正 |

## 📊 系列文章的質量遞推

每個系列有「質量遞推」的責任：

### **「從毒癮到創造者」系列為例**

| 篇章 | 定位 | 質量標準 | 狀態 |
|------|------|---------|------|
| **第一篇**《你不是沉迷..》 | **問題層** - 激發共鳴 | ⭐⭐⭐⭐⭐ 必須爆款 | ✅ 達標（146訂） |
| **第二篇**《你被偷走了什麼》 | **診斷層** - 深化問題 | ⭐⭐⭐⭐⭐ 質量不低 | ✅ 達標（框架清晰） |
| **第三篇**《引力轉移》 | **理論層** - 解釋原理 | ⭐⭐⭐⭐⭐ 最深度 | ✅ 達標（物理學框架） |
| **第四篇**《五分鐘做出..》 | **入口層** - 立即行動 | ⭐⭐⭐⭐ 可實踐 | ⏳ 待發布 |

**規則**：
```
❌ 不允許「序列遞減」
   └─ 第一篇 ⭐⭐⭐⭐⭐，第二篇 ⭐⭐⭐ = 失敗

✅ 應該「質量穩定或遞增」
   └─ 第一篇 ⭐⭐⭐⭐⭐，第二篇 ⭐⭐⭐⭐⭐ = 成功
   └─ 讀者對系列的期待只會上升，不會下降
```

## 🎯 寫作前的自問清單

發布前 24 小時，問自己：

```
□ 「這篇文章有沒有系統性框架？」
  → 如果沒有，不要發。返工。

□ 「我在這篇文章中提出了什麼新觀點？」
  → 如果答不出來，這篇不新。返工。

□ 「讀者讀完這篇會有啟蒙感嗎？」
  → 如果答是「會覺得有趣」而非「思維改變」，質量不夠。

□ 「為什麼這篇值得讀者花 5 分鐘？」
  → 如果理由不充分，延後發布。

□ 「這篇會不會傷害我『優質知識供應商』的品牌？」
  → 如果有疑慮，推遲。寧缺毋濫。
```

## 📈 質量帶來的長期效應

```
高質量內容（持續 3 個月）
    ↓
讀者認知：「AI指揮官的文章值得讀」
    ↓
自然轉化率：2-3%（而非靠引流的 0.5-1%）
    ↓
讀者粘性：高（重複讀者多）
    ↓
口碑傳播：「值得推薦給朋友」
    ↓
訂閱增長：穩定 + 高質（非一時爆發後衰退）
    ↓
商業化基礎：付費轉化率高（願意為質量付費）
```

## 🚨 質量 vs 速度的取舍

### **舊策略（已放棄）**
```
目標：14天增 70% 訂閱（148→250）
做法：8篇文章，密集連發，配套引流
結果：打開率從 40% 降到 23%，轉化率 81% 下滑 ❌
學到：速度換來的是質量和信譽下降
```

### **新策略（確立）**
```
目標：30天增 100% 訂閱（148→300+），質量持續優化
做法：4-5篇系列文章，深度沉澱，質量優先
預期：穩定 35-50% 打開率，轉化率 1.5-2.5%，高粘性讀者
意義：建立「優質知識品牌」，而非「短期爆款」
```
