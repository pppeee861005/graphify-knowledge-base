在平民化多智能體工程課程中，我們針對 delegate_task 常見的失敗情況設計了三條簡單有效的恢復策略。這些策略都已在實際任務中驗證過，可直接複製到你的腳本或 Skill 中使用。

1️⃣ 超出併發數限制（max_concurrent_children）

問題
當一次啟動的子任務數量 > max_concurrent_children（預設 3），delegate_task 會立即失敗並回傳類似：

  Error: Too many concurrent children (4 > 3)

恢復做法

• 將任務拆分：把原本要一次發出的 4 個任務拆成 2+2，或直接改用單任務逐一執行。
• 動態調整：若你知道任務輕量，可臨時把 max_concurrent_children 調高（透過 hermes config set delegation.max_concurrent_children 5），但建議在學習階段保守使用 3，避免資源競爭。

範例程式碼（Python）

  ─ python
  from hermes_tools import delegate_task
  def safe_delegate(batch, role, model, context=""):
      """
      批量執行 delegate_task，自動處理併發數超限。
      batch: List of (goal,) 元組，每個元組代表一個子任務的目標。
      """
      results = []
      # 每次最多 3 個（依照 max_concurrent_children）
      for i in range(0, len(batch), 3):
          sub_batch = batch[i:i+3]
          # 這裡使用併發模式（tasks 陣列）來一次送出最多 3 個
          sub_results = delegate_task(
              tasks=[
                  {"goal": g, "context": context, "role": role,
                   "toolsets": ["terminal", "file"],  # 依需求調整
                   "model": {"provider":"openrouter","model":model}}
                  for g, in sub_batch
              ]
          )
          results.extend(sub_results)
      return results
  # 使用方式：
  goals = [("研究趨勢 A",), ("研究趨勢 B",), ("研究趨勢 C",), ("研究趨勢 D",)]
  research_out = safe_delegate(goals, "researcher", "moonshotai/kimi-k2.6")

2️⃣ 改用純文字搜尋（grep / search_files）而非瀏覽器導航

問題
在需要快速檢查本地檔案或程式碼時，使用 browser_navigate + browser_snapshot 會額外載入頁面、執行 JS，速度慢且易於被反爬機制攔截。

恢復做法

• 優先使用終端機工具：search_files（內建 ripgrep）或直接 grep。
• 只在真正需要互動（填表、點擊按鈕、處理 SPA）時才啟用瀏覽器。

範例：快速找出所有含「錯誤處理」的 Markdown 檔

  ─ bash
  # 終端機一行搞定
  search_files --pattern "錯誤處理" --target content --file_glob "*.md"

或在 Python 腳本中：

  ─ python
  from hermes_tools import search_files
  matches = search_files(pattern="錯誤處理", target="content", file_glob="*.md")
  print(matches)

3️⃣ 後備模型（fallback_providers）機制自動切換

問題
特定模型在某個時段可能因額度用盡、速率限制或區域封鎖而暫時不可用。

恢復做法
在 config.yaml（或透過 hermes config set）設定 fallback_providers，當首選模型失敗時，系統會依序嘗試備援模型，無需在程式碼中加入重複嘗試邏輯。

設定範例（~/.hermes/config.yaml）

  ─ yaml
  llm:
    default_provider: openrouter
    fallback_providers:
      - provider: openrouter
        model: moonshotai/kimi-k2.6          # 先試這個（中文優秀）
      - provider: openrouter
        model: deepseek/deepseek-v4-pro      # 若失敗則切換到這個（推理強）
      - provider: openrouter
        model: anthropic/claude-sonnet-4     # 再試這個（穩定輸出）

在腳本中直接指定備援模型（若不想動態改全域設定）：

  ─ python
  from hermes_tools import delegate_task
  def call_with_fallback(goal, context, role):
      models = [
          "moonshotai/kimi-k2.6",
          "deepseek/deepseek-v4-pro",
          "anthropic/claude-sonnet-4"
      ]
      last_err = None
      for m in models:
          try:
              return delegate_task(
                  goal=goal,
                  context=context,
                  role=role,
                  model={"provider":"openrouter","model":m},
                  timeout=120
              )
          except Exception as e:
              last_err = e
              continue  # 嘗試下一個模型
      # 若全部失敗，拋出最後一次錯誤
      raise last_err
  # 使用方式
  report = call_with_fallback(
      goal="請用三個條列式說明 AI 醫療的主要趨勢",
      context="您是波江星人工程師洛基，偏好繁體中文、直接顯示可複製指令。",
      role="researcher"
  )
