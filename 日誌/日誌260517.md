### 上午工作流程（08:00–12:00）— Kling AI API 集成

**具體行動 1：API 認證方式研究**
- 嘗試 6 種不同的 HTTP 認證方式，全部失敗（401）
  - Bearer + Access Key ❌
  - Bearer + Secret Key ❌
  - Payload 中包含密鑰 ❌
  - AWS Signature V4 ❌
  - Basic Auth ❌
  - 自定義 Header ❌

- **突破點**：GitHub 搜尋發現正確方式
  - 需要用 AccessKey + SecretKey 生成 JWT Token
  - 使用 HMAC SHA256 算法
  - Token payload 必須包含：`iss`、`iat`、`nbf`、`exp`

**具體行動 2：JWT Token 生成實現**
- Python 實現（使用 PyJWT 2.10.1）
- 關鍵發現：
  - `nbf` (not before) 欄位是必需的，否則返回 401
  - Token 有效期 5 分鐘（需要定期更新）
  - Authorization header 格式：`Bearer {token}`

**具體行動 3：API 端點確認**
- 多次測試後確認正確端點：
  - ✅ `POST https://api.klingai.com/v1/videos/text2video`
  - ❌ `https://api.klingai.com/v1/text2video` (404)
  - ❌ `https://api.kling.ai/v1/...` (DNS 不存在)

**具體行動 4：主腳本開發**
