# 為讀者開一扇門 - 完整計劃書

**發起人**：新人類聯盟（豪力風神）  
**核心使命**：知識開採 → 產品產出 → 生態豐富 → 全球共好  
**計畫目標**：讓讀者從消費者變成協作創作者  
**關鍵詞**：遊戲部署、AI 協作、知識複利、生態飛輪

---

## 一、核心概念

### 1.1 為什麼叫「開一扇門」？

不是給讀者一條魚，而是給他們釣竿、教他們釣魚、讓他們教別人釣魚。

```
階段 1：給成果（你的遊戲）
階段 2：打開工具（Claude）
階段 3：讀者自主創作（無限可能）
階段 4：生態複利（讀者成為傳播者）
```

### 1.2 這個計畫解決什麼問題

**傳統知識分享的瓶頸**：
- 讀者是被動消費者（看完就忘）
- 知識衰減率 90%（三天後）
- 無法激發創意
- 分享無法擴展

**新人類聯盟的解決方案**：
- 讀者變成協作者（邊玩邊學邊創造）
- 知識通過創作內化（每改一次就更懂一次）
- 激發創意的基礎是「可改造的成品」
- 每個讀者成為新的傳播節點

## 二、技術架構

### 2.1 遊戲設計的自給自足性

你的遊戲（farmer_survivor_with_music.html）的設計特點：

```
┌─────────────────────────────────┐
│     一個 HTML 檔案              │
├─────────────────────────────────┤
│  • 遊戲邏輯 (JavaScript)        │
│  • 視覺資源 (Canvas API)        │
│  • 音樂資源 (Base64 編碼)       │
│  • 無外部依賴                   │
└─────────────────────────────────┘
      ↓
   完全獨立運行
      ↓
可部署到任何網頁伺服器
      ↓
讀者可用 Claude 即時改造
```

### 2.2 部署架構

#### 選項 A：Vercel（推薦 ⭐⭐⭐）

```
成本：免費
部署時間：5 分鐘
URL 格式：https://yourproject.vercel.app/game.html
優點：自動 HTTPS、CDN 加速、自動部署
缺點：無
適合：快速上線、全球玩家
```

**部署步驟**：
1. 建立 GitHub repo（game-library）
2. 上傳 farmer_survivor_with_music.html
3. 在 vercel.com 連接 repo
4. 一鍵 Deploy
5. 自動獲得 URL

#### 選項 B：GitHub Pages

```
成本：免費
部署時間：10 分鐘
URL 格式：https://yourusername.github.io/games/
優點：完全免費、版本控制
缺點：沒有 CDN（亞太有點慢）
適合：長期穩定分享
```

#### 選項 C：自有伺服器

```
成本：¥ 視伺服器
部署時間：視配置
URL 格式：自訂
優點：完全控制
缺點：需要維護
適合：已有網站的情況
```

**推薦選擇**：Vercel（成本零、速度快、自動化）

### 2.3 讀者改造的技術流程

```
讀者在 claude.ai：
  ↓
「我找到遊戲網址：[URL]」
  ↓
「改成我想要的樣子：[創意描述]」
  ↓
Claude 動作：
  1. 訪問 URL，讀取 HTML
  2. 分析遊戲架構
  3. 識別可改造部分（敵人類型、難度、視覺、規則）
  4. 保留核心邏輯（遊戲迴圈、得分系統）
  5. 輸出新的完整 HTML
  ↓
讀者得到新遊戲代碼
  ↓
複製 → 保存 → 玩 → 分享
```

**為什麼 Claude 能做到**：
- ✅ 能訪問網頁（web browsing）
- ✅ 能讀懂 HTML/JavaScript/Base64
- ✅ 能修改代碼同時保留架構
- ✅ 能根據自然語言快速變異
